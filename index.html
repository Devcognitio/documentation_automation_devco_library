Introducción

El propósito de esta libreria es servirte de ayuda en tu automatización si estas usando el framework de Serenity BDD y Screenplay como patrón de diseño para automatización ya sea para web o móvil.
Puedes leer más acerca de estas tecnologias en los siguientes links:

  • SerenityBDD
  • Screenplay Pattern

Motivation

¿Quien no ha soñado con tener superpoderes?

Sabemos que no tenemos la posibilidad de darte fuerza extra, de volar o hacerte invisible. pero algo que si podemos hacer es darte superpoderes para tu código de automatización.
Asi es. ¿Cuántas veces te ha pasado que cada vez que creas un proyecto de automatización web o móvil, terminas desarrollando nuevas funcionalidades para tus actores? Serenity BDD con Screenplay es una poderosa herramienta que permite desarrollar pruebas altamente legibles y reutilizables. Pero a veces, necesitamos un poco más de habilidades o acciones para hacer que nuestras pruebas se ajusten mucho más flujo que tiene el actor a través de la aplicación. En Devco te damos superpoderes , por lo que hemos desarrollado una libreria que, bien utilizada con Screenplay, hará que cualquier usuario sea invencible en este viaje, ademas puedes usarla en suites de prueba únicas / multiplataforma. Configuración

Esta libreria usa lenguaje Java como base, normalmente usamos herramientas de automatización de tareas como Maven o Gradle para adicionar nuestra dependencia a un proyecto.
Si estas usando Gradle, de esta forma puedes importar nuestra libreria 

                
                    dependencies {
                        implementation group: 'co.com.devco.automation', name: 'automationDevcoLibrary', version: '2.0.2'
                    }

Si estas usando Maven puedes importarla asi: 

                
                  <dependency>
                    <groupId>co.com.devco.automation </groupId>
                    <artifactId>automationDevcoLibrary </artifactId>
                    <version>2.0.2 </version>
                  </dependency>

Estrategia para Locators en Móviles

Locator Strategy

Esta utilidad te permite manejar una automatización móvil multiplataforma (Android e iOS) utilizando el mismo script. La ventaja de utilizar esta estrategia de locators es que puede utilizar locators estáticos y dinámicos fácilmente.

Métodos disponibles para la estrategía Locators en Móviles

Android iOS Tipo
1 withAndroidXpathDynamic() withIosXpathDynamic() Estrategia Dinámica
2 withAndroidXpathStatic() withIosXpathStatic() Estrategia Estática
3 withAndroidId() withIosId() Estrategia Estática
4 withAndroidName() withIosName() Estrategia Estática
5 withAndroidClassName() withIosClassName() Estrategia Estática
6 withAndroidAccesibilityId() withIosAccesibilityId() Estrategia Estática
7 withIosClassChain() Estrategia Estática
8 withIosPredicatedString() Estrategia Estática

Antes de comenzar con cualquier estrategia, lo primero que debe hacer es crear 2 carpetas en su capa de User Interface. Carpeta Page y Locator

Estrategia para Locators Dinámicos

En su carpeta de Locators, creará una clase y uno por uno los objetos tipo Locators que necesita. Utilice la palabra Locator en el nombre de la clase como recomendación para diferenciarla más adelante.


              public class LoginLocator {
                  public static final Locator TXT_USERNAME_LOCATOR =
                          locator()
                                  .withAndroidXpathDynamic("//android.widget.EditText[contains(@resource-id, '{0}')]")
                                  .withIosXpathDynamic("(//XCUIElementTypeStaticText[@name='{0}'])[1]");
              }

Cree en la carpeta Page una clase con el mismo nombre que creó La clase Locator, pero en este caso, debe tener la palabra "Page" en el nombre para diferenciarla. En esta clase crearás tus objetos de tipo Target enviando el Locator como parámetro. Importante, los métodos para llamar al Locator dinámico son locatedBy y se envían como parámetro dynamicElement() Asi:


              public class LoginPage {
                  public static final Target TXT_USERNAME =
                      Target.the("Input to username").locatedBy(dynamicElement(TXT_USERNAME_LOCATOR));
                }

Ahora su automatización está fucnionando con Locators dinámicos usando xpath con el parámetro {0} .... {n}. Cuando invoques al Target en una clase Interaction o Task solo necesita usar el método of() en el Target, enviar el parámetro y que la librería hará el resto.

Estrategia para Locators Estáticos

Para locators estáticos es la misma configuración anterior en la clase Locators


  public class LoginLocator {
    public static final Locator TXT_USERNAME_LOCATOR =
           locator()
                   .withAndroidId("textInput_name")
                   .withIosPredicatedString("label == 'userName'");
}

Algo muy importante es, en tu clase Page el método que debes llamar es located() y el parámetro que este método recibe es otro método llamado theElementBy() y este recibe el objeto Locator creado


  public class LoginPage {
      public static final Target TXT_USERNAME =
          Target.the("Input to username").located(theElementBy(TXT_USERNAME_LOCATOR));
    }

A tener en cuenta:

  • En tu Clase Locator, define todos los elementos que pertenecen a la vista que vas a automatizar. No es una clase por locator.
  • No puedes definir un Locator con una estrategia estática en un SO y dinámica en otro porque la lógica de tu Interaction o tu Task cambia y fallará al ejecutar.
  • Para las estrategias estáticas, cuando crea los objetos de los locators, el método mostrará todas las estrategias que puede usar para un elemento en Android e iOS.

Móviles

En esta sección, encontrará Acciones y Habilidades disponibles para Android e iOS. Puede usar estas acciones en tu automatización de Serenity BDD usando el patrón Screenplay.

Acciones

Por definición general, las acciones realizan toda la interacción directa con la API de Selenium y nos permiten elegir un título más legible para lo que estamos haciendo. Un claro ejemplo de esto es Click.on(), por lo que las acciones que vas a ver aquí están enfocadas a interacciones específicas con los elementos o pantalla para móvil.

Arrastrar Elemento

Drag Element

Arrastrar elemento (Drag Element) es una interacción en la que puede arrastrar "literalmente" elementos específicos en la pantalla de diferentes formas. Esta interacción es útil para deslizar objetos para eliminarlo o ver más opciones, etc.

¿Cómo usarlo?


              Drag.theElement(Target.the()).to(UP)
              Drag.theElement(Target.the()).to(DOWN)
              Drag.theElement(Target.the()).to(LEFT)
              Drag.theElement(Target.the()).to(RIGHT)

Scroll

Scroll

Scroll es una excelente interacción que te ayuda a moverse sobre la pantalla hasta encontrar los elementos que necesites y están en la parte inferior o incluso en la parte superior. Sí, funciona en ambos sentidos, además puedes enviar como parámetro la cantidad máxima de intentos que vas a hacer para encontrar el elemento, en caso de que no lo encuentres continúa con tu prueba según corresponda.

¿Cómo usarlo?

Para realizar un scroll hacia abajo, usa:


            Scroll.untilVisibleTarget(Target).toBottom().untilMaxAttempts(Integer)

Para realizar un scroll hacia arriba, usa:


          Scroll.untilVisibleTarget(Target).toTop().untilMaxAttempts(Integer)

Habilidades

La habilidad es la capacidad que tiene un actor de ejecutar acciones para realizar las tareas asignadas. La habilidad es diferente de una Interacción ya que esta es más técnica, manejar configuraciones en el dispositivo, trabajar con el contexto, etc.

Cambiar Contexto del Driver

Change Driver Context

When you are working with a web app or hybrid app, sometimes you need to change the context because the elements change and the way how you interact too, so if this is your case you need to switch between the Web and native views to WEBVIEW or NATIVE context.

¿Cómo usarlo?

Usa la siguiente habilidad para cambiar a un contexto de tipo webview:


              ChangeDriverContext.toWebview()

Usa la siguiente habilidad para cambiar a un contexto de tipo nativo:


              ChangeDriverContext.toNative()

Elegir una App desde Segundo Plano

Choose App from Background

Esta habilidad permite elegir una aplicación que se está ejecutando en segundo plano y poder utilizarla. Solo necesitas enviar el nombre de la aplicación que se está ejecutando en segundo plano y listo. Importante si la aplicación no está abierta, la habilidad no funcionará.

¿Cómo usarlo?


              ChooseApp.fromBackground("App Name")

Ejecutar la aplicación en Segundo Plano

Run App in Background

Esta habilidad ejecuta la aplicación actual en segundo plano durante un breve período de tiempo y luego vuelve a la aplicación como aplicación principal. Eso ayuda a simular una actualización, un comportamiento específico de su escenario de prueba, o ayuda a que algunos elementos aparezcan en el XML para interactuar con ellos.

¿Cómo usarlo?


              RunAppInBackground.modeForASecond()

Ocultar teclado

Hide Keyboard

A veces, el teclado oculta algunos elementos con los que necesita interactuar. Esta sencilla habilidad ayuda a ocultar el teclado para ver todos los elementos de la vista.
Es solamente para Android.

¿Cómo usarlo?


                Hide.theKeyboard()

Web

En esta sección, encontrará Acciones y Habilidades disponibles para las páginas web. Estan enfocadas para el uso con Serenity BDD y el patrón Screenplay.

Habilidades

La habilidad es la capacidad que tiene un actor de ejecutar acciones para realizar las tareas asignadas.

Cambiar de Pestaña

Change Tab

Algunas páginas web abren nuevas pestañas cuando una interacción hace algo con un elemento. Si su automatización necesita continuar la prueba con la nueva pestaña, debe hacer el cambio, pero el driver se mantiene enfocado en la página actual. Esta habilidad ayuda a cambiar fácilmente entre pestañas.

¿Cómo usarlo?


                ChangeToTab.withIndex(Integer)

Abrir Nueva Pestaña

Open New Tabs

Esta habilidad te permite abrir una nueva pestaña si lo requieres dentro de tu prueba.

¿Cómo usarlo?


                OpenNewTab.onBrowser()

Esperas

En esta sección vas a encontrar un Wait explicito específico.

Tiempo de Espera

Stand By

Esta espera explícita es útil para esperar un elemento sobre le cual tenemos que indicar el tiempo de espera usando la visibilidad como condición para continuar.

¿Cómo usarlo?


                StandBy.visibilityOf(Target).forNoMoreThan(Integer).seconds

Utilidades

En esta sección, encontrará algunas utilidades para manejar archivos externos. Estas utilidades no están construidas bajo el patrón Screenplay, sin embargo es útil para tu automatización.

Leer Datos desde un CSV

Read Data from CSV

En un script de automatización, podemos administrar los datos con múltiples estrategias, que incluyen archivos externos como .xlsx o .csv. Esta utilidad lee un archivo CSV y extrae los datos que necesita utilizando el nombre de una columna como filtro.


Como recomendación, tu archivo debe ser llamado con la palabra data_ como prefijo y luego el complemento, por ejemplo, data_login.csv y el archivo debe estar localizado en una carpeta "data" en la capa de recursos en test.

¿Cómo usarlo?


            DataCSV.getDataCsv(fileName, filter)

Administrador de PDF

PDF Manager

Esta es una utilidad para leer un archivo PDF desde una URL en el navegador o una ruta específica en su computadora, además, captura el contenido del archivo con el fin de que pueda manipular la información.

¿Cómo usarlo?

Leer un PDF desde el navegador enviando la URL como parámetro

            ReadPdf.readPdfFromBrowser()
Leer un PDF desde el sistema enviando la ruta como parámetro

            ReadPdf.readPdfFromFile()
Obtener la información de un PDF

            ReadPdf.getPdfLines()
